# Project 01 - SauceDemo Login Test Cycle

## Overview
This project is a QA test cycle for login validation on the SauceDemo application. It includes both positive and negative test scenarios to verify authentication behavior, input validation, and error handling.

---

## Application Under Test
- **Name:** SauceDemo
- **URL:** https://www.saucedemo.com
- **Module:** Login Functionality

---

## Objective
To validate the login feature by executing structured test cases covering:
- Valid login behavior
- Invalid credential handling
- Empty field validation
- Locked-out user behavior

---

- Observed system behavior

## Scope Coverage 
### Positive Testing
- Valid user login with correct credentials
- Successful navigation to inventory page after authentication

### Negative Testing
- Invalid password validation
- Empty username validation
- Empty password validation
- Locked-out user validation

This test cycle evaluates the core login functionality of the SauceDemo application using structured manual QA testing. All test cases were executed and documented, with issues tracked and managed through Jira where applicable.
